## SECTION 2: ON-PAGE SEO AUDIT — LANDING PAGE

**File:** `/Users/jonberard/Desktop/smalltalk/app/page.tsx` (1,020 lines)

### 2.1 Critical Issue: "use client" Directive

**Line 1:** `"use client";`

This is the single biggest SEO problem on the site. The entire landing page is a client component. This means:
- The page is rendered entirely in the browser via JavaScript
- Googlebot renders JS, but with delays, reduced crawl budget, and known inconsistencies
- The full page content is NOT available in the initial HTML response
- The JSON-LD structured data is injected client-side, which Google may or may not pick up
- All text content depends on JavaScript execution

**Impact:** SEVERE. Google can render JS but deprioritizes it. For a new domain with zero authority, every disadvantage compounds.

## SECTION 4: TECHNICAL SEO AUDIT

### 4.1 Core Web Vitals Concerns

**Largest Contentful Paint (LCP):**
- RISK: HIGH. The entire page is "use client" (line 1 of page.tsx). This means:
  - Browser downloads JS bundle
  - React hydrates
  - Content renders
  - LCP fires only after all this completes
- The landing page includes heavy interactive components: `LandingDemo` (631 lines of animation code), `ReplyDemo` (auto-cycling word-by-word typing animation), `HeroPhone` (floating animation)
- No `<img>` tags means LCP is likely a text block, which depends on font loading

**First Input Delay / Interaction to Next Paint (INP):**
- Multiple `useEffect` hooks with `setTimeout`/`setInterval` running simultaneously
- `ReplyDemo` has continuous timer cycles
- `LandingDemo` has a phase-based animation loop with multiple concurrent timers
- These shouldn't block INP directly but add JS main thread pressure

**Cumulative Layout Shift (CLS):**
- RISK: MEDIUM. The `useFadeUp()` hook (line 12-37) sets `opacity: 0` on all `[data-fade]` elements, then animates them in via IntersectionObserver. If elements don't have explicit dimensions, this can cause layout shifts as content appears.
- Font loading: Fonts use `display: "swap"` (layout.tsx lines 9, 16, 23), which means FOIT -> FOUT transition can cause shifts. However, this is the correct approach.

**Recommendation:** Convert the landing page to a Server Component. Extract only the interactive parts (`LandingDemo`, `ReplyDemo`, the auth-aware CTA buttons) into client components. The rest (headings, text, FAQ, pricing) should be server-rendered.
